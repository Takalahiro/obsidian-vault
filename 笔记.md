# 【外包】Obsidian 笔记 → Astro 静态网站（地基部分）

## 项目简介

我有一个 Obsidian 笔记库，想做成公开的个人知识网站。**只外包"笔记 → 网页"
的渲染部分**，上层的 AI 问答、知识图谱、Dashboard 等功能我自己开发。

需要你做的，简单说就是：**给我一个稳定的 Astro 项目，能把我所有 Obsidian
笔记正确显示成网页**。

## 技术栈（必须）

- Astro 5（最新版）
- pnpm（包管理器）
- Tailwind CSS v4
- Svelte 5（如有交互组件）
- TypeScript 严格模式
- 部署目标：Cloudflare Pages

## 必须支持的 Obsidian 语法

1. 标准 Markdown（标题、粗斜体、列表、引用、分隔线等）
2. YAML Frontmatter 解析
3. 双链 `[[note]]`
4. 双链带别名 `[[note|显示文本]]`
5. 双链跳锚点 `[[note#heading]]`
6. 图片嵌入 `![[image.png]]`
7. 笔记嵌入 `![[other note]]`
8. 标签 `#tag` 和嵌套标签 `#area/study`
9. Callouts：`> [!note]` `> [!warning]` `> [!info]` `> [!tip]` `> [!danger]` 等
10. 代码块（带语法高亮，至少支持 python/ts/js/bash/json）
11. KaTeX 数学公式（行内 + 块）
12. Mermaid 图表
13. 任务列表 `- [ ]`
14. 表格（含对齐）
15. 脚注 `[^1]`
16. 高亮 `==text==`
17. 删除线 `~~text~~`

## 必须实现的网站功能

1. 动态路由（访问 `/path/to/note` 显示对应笔记）
2. 首页笔记列表（按修改时间排序）
3. 反向链接（笔记底部显示"被哪些笔记引用"）
4. 目录大纲（TOC，显示当前页面所有标题）
5. 标签页（`/tags/<tag>` 列出该标签下所有笔记）
6. 断链检测（构建时打印警告，但不报错）
7. 404 页（找不到笔记时显示友好的 404）
8. 响应式（手机、平板、桌面正常显示）
9. 暗色 / 浅色模式切换
10. 基础站内搜索（按标题 + 内容搜索）

## 代码质量要求

- TypeScript 严格模式 `strict: true`
- 关键函数有 JSDoc 注释
- 笔记解析逻辑独立成模块（`src/lib/obsidian/`）
- 至少 5 个核心函数有 Vitest 单元测试
- `pnpm build` 必须无 error、无 warning
- Lighthouse 分数 ≥ 90（性能 / 可访问性 / SEO）
