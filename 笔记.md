## 🟢 Prompt 4：Wiki Links + Callouts（Obsidian 特有语法）

```

</div>

# 🎮 使用技巧 + 避坑指南

## ⭐ Top 5 技巧

### 1. **不要一次喂多条 prompt**
每条 prompt 等 AI 真的做完、你验收通过，再喂下一条。否则容易堆 bug。

### 2. **遇到报错直接复制贴回去**
```
我跑了 pnpm test，报这个错：
[粘贴完整错误信息]
请修复。
```
AI 修 bug 比写新代码强 10 倍。

### 3. **每个 Prompt 结束都让 AI commit**
```
请把当前进度 git commit，commit message 用 "feat: 完成 wiki link 解析"
```
这样你随时能回滚。

### 4. **看不懂的代码立刻问**
```
请用大白话解释一下 `remarkWikiLinks` 这个函数怎么工作的，
当作给一个大一学生讲。
```
你才大一，**学习的机会比代码本身值钱**。

### 5. **定期让 AI 自查**
做完每个 Prompt 后：
```
请检查刚才你写的代码，看看有没有：
1. TypeScript 类型不严格的地方
2. 没处理的边缘情况
3. 可以提取的重复代码
列出问题，但不要修，先让我看看。

## ⚠️ Top 5 坑

### 1. ❌ AI 会偷偷换技术栈
**症状**：你说 Tailwind v4，它给你装 v3（因为它训练数据里 v3 更多）  
**对策**：每个 Prompt 都强调"严格用 v4"，做完后跑 `cat package.json` 检查版本

### 2. ❌ 测试是"假"通过的
**症状**：测试用 `expect(true).toBe(true)` 这种废话填  
**对策**：让 AI 把测试代码贴出来给你看，你哪怕看不懂也扫一眼

### 3. ❌ "完成了"其实没完成
**症状**：AI 说"我已经实现了 XXX"，但你跑 `pnpm dev` 报错  
**对策**：每个 Prompt 都要求"跑一遍验收命令并把输出贴给我"

### 4. ❌ 不停往项目里塞新依赖
**症状**：装了 20 个奇怪的包，build 越来越慢  
**对策**：定期问"列出 package.json 里所有依赖并解释每个是干嘛的"

### 5. ❌ 上下文丢失
**症状**：聊到 Prompt 5 时，AI 忘了 Prompt 2 里定义的类型  
**对策**：每条 Prompt 开头加一句"参考 src/lib/obsidian/types.ts 里的类型定义"

  import { onMount } from 'svelte';

  let isDark = $state(false);

  onMount(() => {
    isDark = document.documentElement.classList.contains('dark');
  });

  function toggle() {
    isDark = !isDark;
    if (isDark) {
      document.documentElement.classList.add('dark');
      localStorage.setItem('theme', 'dark');
    } else {
      document.documentElement.classList.remove('dark');
      localStorage.setItem('theme', 'light');
    }
  }

### Step 1-H：工具函数 — 反链扫描

创建 `src/lib/backlinks.ts`：

```ts
import { getCollection, type CollectionEntry } from 'astro:content';

export type Note = CollectionEntry<'notes'>;

// 从笔记 body 里抽出所有 [[wiki link]] 目标
function extractWikiLinks(body: string): string[] {
  const re = /$$\[([^$$|#]+)(?:[#|][^\]]*)?\]\]/g;
  const links: string[] = [];
  let m: RegExpExecArray | null;
  while ((m = re.exec(body)) !== null) {
    links.push(m[1].trim().replace(/ /g, '-').toLowerCase());
  }
  return links;
}

// 把笔记的 id（路径）规范化为 slug
export function noteIdToSlug(id: string): string {
  // 去掉 .md，把空格替换为 -，全转小写
  return id
    .replace(/\.md$/, '')
    .split('/')
    .pop()!
    .replace(/ /g, '-')
    .toLowerCase();
}

// 拿到某篇笔记的所有反向链接
export async function getBacklinks(currentSlug: string): Promise<Note[]> {
  const all = await getCollection('notes', ({ data }) => !data.draft);
  const backlinks: Note[] = [];
  for (const note of all) {
    if (noteIdToSlug(note.id) === currentSlug) continue;
    const targets = extractWikiLinks(note.body ?? '');
    if (targets.includes(currentSlug)) {
      backlinks.push(note);
    }
  }
  return backlinks;
}
```

          <a href={`/notes/${slug}`} class="block p-4 rounded-lg border border-gray-200 dark:border-gray-800 hover:border-blue-500 transition">
            <h2 class="font-semibold">{title}</h2>

</BaseLayout>
```

### Step 1-N：处理 Obsidian 图片嵌入

`![[image.png]]` 这种语法需要把 obsidian-vault 里的图片暴露给 Astro。最简单的方案：把 obsidian-vault 里所有图片在构建时复制到 `public/` 下。

创建 `scripts/sync-assets.mjs`：

```js
import { cp, mkdir, readdir, stat } from 'node:fs/promises';
import { join, extname } from 'node:path';

const VAULT = './obsidian-vault';
const TARGET = './public/vault-assets';
const EXTS = new Set(['.png', '.jpg', '.jpeg', '.gif', '.webp', '.svg', '.pdf']);

async function walk(dir, base = dir) {
  const entries = await readdir(dir, { withFileTypes: true });
  for (const e of entries) {
    if (e.name.startsWith('.')) continue;
    const full = join(dir, e.name);
    if (e.isDirectory()) {
      await walk(full, base);
    } else if (EXTS.has(extname(e.name).toLowerCase())) {
      const rel = full.replace(base, '').replace(/^[\\/]/, '');
      const dest = join(TARGET, rel);
      await mkdir(dest.substring(0, dest.lastIndexOf(/[\\/]/.test(dest) ? dest.match(/[\\/]/)[0] : '/')), { recursive: true }).catch(() => {});
      await cp(full, dest, { recursive: false });
    }
  }
}

await mkdir(TARGET, { recursive: true });
await walk(VAULT);
console.log('✅ Assets synced to', TARGET);
```

修改 `package.json` 的 `scripts`：

```json
"scripts": {
  "dev": "pnpm sync && astro dev",
  "build": "pnpm sync && astro build",
  "sync": "node scripts/sync-assets.mjs",
  "preview": "astro preview",
  "astro": "astro"
}
```

并把 `![[xxx.png]]` 转换为 `<img src="/vault-assets/xxx.png">` 的逻辑通过自定义 remark 插件实现。创建 `src/lib/remark-obsidian-image.mjs`：

```js
import { visit } from 'unist-util-visit';

export default function remarkObsidianImage() {
  return (tree) => {
    visit(tree, 'text', (node, index, parent) => {
      if (!node.value || !parent || index === null) return;
      const re = /!$$\[([^$$]+)\]\]/g;
      if (!re.test(node.value)) return;
      re.lastIndex = 0;

      const parts = [];
      let last = 0;
      let m;
      while ((m = re.exec(node.value)) !== null) {
        if (m.index > last) {
          parts.push({ type: 'text', value: node.value.slice(last, m.index) });
        }
        const filename = m[1].split('|')[0].trim();
        parts.push({
          type: 'image',
          url: `/vault-assets/${filename}`,
          alt: filename,
        });
        last = m.index + m[0].length;
      }
      if (last < node.value.length) {
        parts.push({ type: 'text', value: node.value.slice(last) });
      }
      parent.children.splice(index, 1, ...parts);
    });
  };
}
```

需要先安装 `unist-util-visit`：
```powershell
pnpm add unist-util-visit
```

然后在 `astro.config.mjs` 的 `remarkPlugins` 数组**最前面**加入：
```js
import remarkObsidianImage from './src/lib/remark-obsidian-image.mjs';
// ...
remarkPlugins: [
  remarkObsidianImage,  // 放最前，先处理图片
  [remarkWikiLink, { ... }],
  remarkObsidianCallout,
],
```
