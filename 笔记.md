## 🟢 Prompt 4：Wiki Links + Callouts（Obsidian 特有语法）

```

</div>

# 🎮 使用技巧 + 避坑指南

## ⭐ Top 5 技巧

### 1. **不要一次喂多条 prompt**
每条 prompt 等 AI 真的做完、你验收通过，再喂下一条。否则容易堆 bug。

### 2. **遇到报错直接复制贴回去**
```
我跑了 pnpm test，报这个错：
[粘贴完整错误信息]
请修复。
```
AI 修 bug 比写新代码强 10 倍。

### 3. **每个 Prompt 结束都让 AI commit**
```
请把当前进度 git commit，commit message 用 "feat: 完成 wiki link 解析"
```
这样你随时能回滚。

### 4. **看不懂的代码立刻问**
```
请用大白话解释一下 `remarkWikiLinks` 这个函数怎么工作的，
当作给一个大一学生讲。
```
你才大一，**学习的机会比代码本身值钱**。

### 5. **定期让 AI 自查**
做完每个 Prompt 后：
```
请检查刚才你写的代码，看看有没有：
1. TypeScript 类型不严格的地方
2. 没处理的边缘情况
3. 可以提取的重复代码
列出问题，但不要修，先让我看看。

## ⚠️ Top 5 坑

### 1. ❌ AI 会偷偷换技术栈
**症状**：你说 Tailwind v4，它给你装 v3（因为它训练数据里 v3 更多）  
**对策**：每个 Prompt 都强调"严格用 v4"，做完后跑 `cat package.json` 检查版本

### 2. ❌ 测试是"假"通过的
**症状**：测试用 `expect(true).toBe(true)` 这种废话填  
**对策**：让 AI 把测试代码贴出来给你看，你哪怕看不懂也扫一眼

### 3. ❌ "完成了"其实没完成
**症状**：AI 说"我已经实现了 XXX"，但你跑 `pnpm dev` 报错  
**对策**：每个 Prompt 都要求"跑一遍验收命令并把输出贴给我"

### 4. ❌ 不停往项目里塞新依赖
**症状**：装了 20 个奇怪的包，build 越来越慢  
**对策**：定期问"列出 package.json 里所有依赖并解释每个是干嘛的"

### 5. ❌ 上下文丢失
**症状**：聊到 Prompt 5 时，AI 忘了 Prompt 2 里定义的类型  
**对策**：每条 Prompt 开头加一句"参考 src/lib/obsidian/types.ts 里的类型定义"
